Windows Backup and Restore

Please note that a restore can NOT be done while the mmria-server.exe is running.

Unlike a restore a backup can be done while the mmria-server.exe is running.

To perform backup or restore you'll need 4 pieces of information:

  • 1.user <- database administrator userid
  • 2.password <- database administrator password
  • 3.database_url <- couchdb url
  • 4.backup_file_path <- location where you wish to save data

Please note that 1 database is involved in backing up and restoring:

  • mmrds

To perform a backup:

example:
mmria.exe backup user_name:[admin_user] password:[admin_password] database_url:http://localhost:5984/mmrds backup_file_path:c:\temp\bk-mmrds.bk

To perform a restore:

Performing a Restore requires the following steps:

  • 1 stop the mmria-server.exe
  • 2 delete the database that is to be restored
  • 3 manually create database in CouchDB UI
  • 4 use mmria.exe to restore database
  • 5 apply appropriate role permission in CouchDB UI
example:


mmria.exe restore user_name:[admin_user] password:[admin_password] database_url:http://localhost:5984/mmrds backup_file_path:c:\temp\bk-mmrds.bk

Permissions

After doing a restore please ensure the following perissions are defined on the appropriate database.



mmrds 
   admin = form_designer
  members = abstractor + data_analyst + timer
